## 🎯 О чём этот проект?

Это исследование поведения пользователей в мобильном приложении для продажи ненужных вещей (аналог Avito). Заказчик хочет понять, как пользователи взаимодействуют с приложением, выделить группы с разным поведением и дать рекомендации для развития продукта.

### Ключевые вопросы исследования:
1. Как пользователи ведут себя в приложении? (retention, время, частота действий, конверсия)
2. Можно ли выделить группы пользователей с разным поведением?
3. Какие группы пользователей важнее для бизнеса?
4. Влияют ли рекомендации на конверсию?
5. Различается ли поведение пользователей из разных источников установки?

## 🔍 Что мы делали шаг за шагом?

### Шаг 1. Подготовка данных
- Загрузили библиотеки (pandas, numpy, matplotlib, seaborn, scipy, plotly)
- Открыли файлы: 4293 пользователя, 74 197 событий
- **Пропуски** — отсутствуют
- **Привели названия столбцов** к единому формату (user_id, event_time, event_name)
- **Объединили события:**
  - search_1 — search_7 → search_other
  - contacts_show и show_contacts → contacts_show
- **Типы данных:** event_time приведён к datetime64[ns]
- **Дубликаты:** удалили 1190 явных дубликатов (1,6% от всех событий)
- **Период данных:** 7 октября — 3 ноября 2019 (28 дней)
- **Распределение по источникам:**
  - yandex: 1 934 (45,05%)
  - other: 1 230 (28,65%)
  - google: 1 129 (26,29%)

### Шаг 3. Сегментация пользователей (по дням недели установки)

#### Retention rate по группам
**Лидеры:** пользователи, установившие приложение в **среду** — чаще возвращаются

#### Распределение сессий
- Пользователи из группы "среда" имеют наименьшее количество нулевых сессий
- По количеству не-нулевых действий все группы показывают идентичные результаты

#### Конверсия по группам

| Tuesday | 11 413 | 35,48% |

**Вывод:** Пользователи, установившие приложение во вторник, чаще совершают целевое действие

#### Итог по сегментации:
- **Чаще возвращаются:** пользователи из группы "среда"
- **Чаще совершают целевое действие:** пользователи из группы "вторник"

### Шаг 4. Проверка гипотез

#### Гипотеза 1
> Пользователи из разных источников (yandex vs google) демонстрируют разную конверсию

**Результат:** ❌ **Статистически значимой разницы не обнаружено**

Однако обнаружены различия в других действиях:
- tips_show (увидел рекомендации)
- search_other (поисковые запросы)
- photos_show (просмотр фото)

#### Гипотеза 2

**Результат:** ✅ **Статистически значимая разница есть**

Пользователи, взаимодействующие с рекомендациями, показывают более высокую конверсию в просмотр контактов.

## 📊 Общие выводы

### 1. Поведение пользователей
- **Основной сценарий:** пользователи видят много рекомендаций (54% всех действий)
- **Проблема:** низкий клик по рекомендациям (всего 0,7% от действий)
- **Целевое действие:** просмотр контактов составляет 6,4% от всех действий
- **Удержание:** резко падает после 2-й недели использования

### 2. Сегментация
| Характеристика | Какая группа лучше |
|----------------|-------------------|
| Retention | Среда |
| Конверсия | Вторник |

### 3. Гипотезы
- **Источник установки не влияет** на конверсию
- **Рекомендации работают** — пользователи, видящие их, имеют более высокую конверсию

## 💡 Рекомендации для бизнеса

### 1. Мотивация для возврата
- Пользователи лучше всего удерживаются, если установили приложение в среду
- Нужно разработать механики для повышения retention: пуш-уведомления, бонусы за возврат, email-рассылки

### 2. Работа с рекомендациями
- **Слишком много показов** — 54% всех действий — это потенциально заспамливает пользователей
- **Низкий клик** — всего 0,7% пользователей кликают по рекомендациям
- **Но рекомендации работают** — пользователи, видящие их, имеют более высокую конверсию
- **Рекомендация:** оптимизировать алгоритм рекомендаций, уменьшить количество показов, повысить релевантность

### 3. Техническая проверка
- Аномально длинные сессии (>80 минут) и большое количество tips_show требуют проверки
- Возможны технические ошибки в логировании или проблемы с API

### 4. Дальнейшие шаги
- Провести A/B-тест: разные алгоритмы рекомендаций vs текущая версия
- Исследовать, почему пользователи из разных дней недели показывают разное поведение
- Углубить анализ "нулевых" сессий — возможно, проблема в первом впечатлении
