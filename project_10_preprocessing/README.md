## 🎯 О чём этот проект?

Это исследование базы данных книжного сервиса по подписке. Компания приобрела крупный сервис для чтения книг и необходимо проанализировать имеющиеся данные для формирования ценностного предложения для нового продукта.

### Что мы хотим узнать?

1. **Сколько книг вышло после 2000 года?** — оценка современного контента
2. **Какие книги получают больше всего обзоров и высокие оценки?** — выявление популярных книг
3. **Какое издательство выпустило больше всего книг (более 50 страниц)?** — ключевые партнёры
4. **Какой автор имеет самую высокую среднюю оценку?** — авторы, на которых стоит делать акцент
5. **Сколько в среднем обзоров оставляют активные пользователи?** — поведение ключевой аудитории

## 📁 Откуда данные?

База данных состоит из 5 таблиц:

### Таблица `books`
| Поле | Что означает |
|------|--------------|
| `book_id` | Уникальный идентификатор книги |
| `author_id` | Идентификатор автора |
| `title` | Название книги |
| `num_pages` | Количество страниц |
| `publication_date` | Дата публикации |
| `publisher_id` | Идентификатор издателя |

### Таблица `authors`
| Поле | Что означает |
|------|--------------|
| `author_id` | Уникальный идентификатор автора |
| `author` | Имя автора |

### Таблица `publishers`
| Поле | Что означает |
|------|--------------|
| `publisher_id` | Уникальный идентификатор издательства |
| `publisher` | Название издательства |

### Таблица `ratings`
| Поле | Что означает |
|------|--------------|
| `rating_id` | Уникальный идентификатор оценки |
| `book_id` | Идентификатор книги |
| `username` | Имя пользователя |
| `rating` | Оценка (числовое значение) |

### Таблица `reviews`
| Поле | Что означает |
|------|--------------|
| `review_id` | Уникальный идентификатор обзора |
| `book_id` | Идентификатор книги |
| `username` | Имя автора обзора |
| `text` | Текст обзора |
